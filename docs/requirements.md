# 프로젝트 요구사항

## 프로젝트 개요
기술사 시험 준비를 위한 서브노트 관리 및 조회 시스템

## 사용자 유형
1. **수강생**: 서브노트를 조회하는 사용자
2. **관리자**: 서브노트를 작성/관리하는 사용자

## 주요 기능
- [x] 인증/권한 시스템
- [x] 노션 레벨 마크다운 에디터
- [x] 트리 구조 카테고리 시스템
- [x] 템플릿 관리 시스템
- [x] 버전 관리 시스템
- [ ] 서브노트 CRUD
- [ ] 검색 및 필터링
- [ ] 카테고리 관리 (별도 메뉴)
- [ ] 템플릿 관리 (별도 메뉴)

## 인증 시스템
- **OAuth 제공자**: Google, Apple
- **필수**: 수강생도 로그인 필요 (관리자 승인 후 접근 가능)

## 서브노트 에디터
- **형식**: 노션 레벨 마크다운
- **저장**: 수동 저장 (실시간 저장 X)
- **기능**:
  - 리치 텍스트 편집
  - 마크다운 문법 지원
  - 코드 블록
  - 이미지 삽입

## 카테고리 시스템
- **구조**: 트리 구조 (무한 depth 지원)
- **관리**: 관리자 화면 별도 메뉴에서 CRUD
- **기능**:
  - 카테고리 생성/수정/삭제
  - 드래그 앤 드롭으로 순서 변경
  - 부모-자식 관계 설정
  - 카테고리별 서브노트 연결

## 템플릿 시스템
- **관리**: 관리자 화면 별도 메뉴에서 CRUD
- **사용**: 서브노트 작성 시 템플릿 선택하여 불러오기
- **기능**:
  - 템플릿 생성/수정/삭제
  - 템플릿명, 설명 관리
  - 노션 레벨 마크다운 형식으로 작성
  - 서브노트 작성 시 템플릿 목록에서 선택

## 버전 관리 시스템
- **대상**: 서브노트 수정 이력
- **기능**:
  - 모든 수정 이력 자동 저장
  - 버전별 미리보기
  - 두 버전 간 비교 (Diff)
  - 이전 버전으로 복구
  - 수정 일시 및 작성자 기록
- **권한**: 관리자만 접근 가능

## 미결정 사항
- 이미지 저장 방식 (Supabase Storage 등)
