# 템플릿 관리 시스템

## 개요
관리자가 사전에 서브노트 템플릿을 작성하고, 새 서브노트 작성 시 템플릿을 불러와 사용할 수 있는 기능

## 템플릿 구조

### 데이터베이스 구조 예시
```sql
templates (
  id: UUID (Primary Key)
  name: VARCHAR (템플릿명)
  description: TEXT (템플릿 설명)
  content: JSONB/TEXT (노션 레벨 마크다운 형식)
  created_by: UUID (Foreign Key -> users.id)
  created_at: TIMESTAMP
  updated_at: TIMESTAMP
)
```

## 관리 기능 (CRUD)

### 1. 생성 (Create)
- 템플릿명 입력
- 템플릿 설명 입력
- 노션 에디터로 템플릿 내용 작성
- 저장

### 2. 조회 (Read)
- 전체 템플릿 목록 조회
- 템플릿 상세 조회
- 템플릿 미리보기

### 3. 수정 (Update)
- 템플릿명 수정
- 템플릿 설명 수정
- 템플릿 내용 수정

### 4. 삭제 (Delete)
- 템플릿 삭제
- 삭제 확인 다이얼로그

## 템플릿 사용 시나리오

### 서브노트 작성 시
1. "새 서브노트" 버튼 클릭
2. 옵션 선택:
   - **빈 문서로 시작**
   - **템플릿에서 시작**
3. 템플릿 선택 시:
   - 템플릿 목록 모달 표시
   - 템플릿 미리보기
   - 템플릿 선택
4. 선택한 템플릿 내용이 에디터에 로드됨
5. 내용 수정 후 저장

## 템플릿 예시

### 기술 개념 템플릿
```markdown
# [기술명]

## 개념
[기술에 대한 간단한 정의]

## 특징
- 특징 1
- 특징 2
- 특징 3

## 장단점
### 장점
- 장점 1
- 장점 2

### 단점
- 단점 1
- 단점 2

## 적용 사례
[실제 적용 사례]

## 참고자료
- [참고자료 링크]
```

### 문제 해결 템플릿
```markdown
# [문제 제목]

## 문제 정의
[문제가 무엇인지]

## 원인 분석
[문제의 원인]

## 해결 방안
### 방안 1
[설명]

### 방안 2
[설명]

## 최적 해결책
[선택한 해결책과 이유]

## 결론
[요약]
```

## UI/UX

### 템플릿 관리 화면
- 템플릿 목록 (카드 형태 또는 테이블)
- 검색 기능
- 새 템플릿 생성 버튼
- 각 템플릿마다:
  - 편집 버튼
  - 삭제 버튼
  - 미리보기 버튼

### 템플릿 선택 모달
- 템플릿 목록 그리드
- 각 템플릿:
  - 템플릿명
  - 설명
  - 미리보기 썸네일
  - 선택 버튼
- 검색 필터

## 추가 기능 (선택사항)
- 템플릿 카테고리 분류
- 자주 사용하는 템플릿 즐겨찾기
- 템플릿 공유 기능
- 템플릿 복제 기능
